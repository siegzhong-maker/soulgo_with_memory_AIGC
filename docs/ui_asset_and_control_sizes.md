# IP 伴游宠物 UI 控件尺寸与素材规范（画师稿件基线）

> 本轮改版范围：**只需要调整「橱柜」和「日记」相关控件的素材与尺寸，其它页面与控件保持现状不动。**
>
> 基于当前 `prototype.html` 原型和 `PRD.md`，统一以 **414 × 896**（接近 iPhone 11/13 Pro Max）为 1× 画板尺寸。画师按 2× / 3× 比例导出即可（如 828×1792、1242×2688）。

## 1. 通用约定

- **整体画板**
  - 画板尺寸：414 × 896（1×）。
  - App 可视区域：与画板一致，顶部状态栏和底部安全区由前端留出，不需要单独切图。
  - PC 预览时，外层会加“手机壳”边框和圆角，这部分无需出图。
- **导出倍数**
  - 建议出 **2× 与 3×** 两套切图，命名后缀如 `@2x`、`@3x`。
  - 文档内所有尺寸以 **1× 像素**标注，画师按倍数放大导出即可。
- **风格约束（与 PRD 第 9 章一致）**
  - 色彩：低饱和、暖色系，治愈卡通风。
  - 组件：大圆角（≥16px）、柔和阴影、新拟物轻量质感。

## 2. 首页地图视图（view-home）

### 2.1 地图区域

- **地图卡片容器 `#china-map-container`**
  - 宽度：全宽减内边距，约 **366 × 260–300 px**。
    - 计算：414 − 2 × 24（外层 `p-6` 内边距）= 366。
  - 高度：前端 `min-height: 200px`，实际视觉预期 **260–300px**（可让插画在纵向略有余量）。
  - 圆角：约 **40px**。
  - 边框：约 **6px** 宽浅色描边。
  - 阴影：柔和下投影，可按常规卡片风格出一张背景九宫格（可选）。

- **中国地图插画（本地资源 `assets/map/china-kawaii-map.png`）**
  - 用途：承载打卡图钉的主背景。
  - 显示方式：在 `366 × 260–300` 的容器内 `object-contain`，上下左右留 12px 内边距。
  - 建议原始尺寸：**至少 1920 × 1080**，16:9 横版。
  - 重要：轮廓清晰，省份边界不必严格准确，但需要有足够留白给图钉。

### 2.2 打卡输入区

- **文本输入框 `#location-input`**
  - 逻辑样式：`py-3 px-4 rounded-2xl`。
  - 建议 1× 视觉尺寸：
    - 高度：**44–48px**（含内边距）。
    - 圆角：**16px**。
    - 左右内边距：各 **16px**。
  - 宽度：剩余空间（414 − 左右 24 内边距 − 右侧按钮宽度 ~ 100px），保证能容纳至少 **8–12 个汉字**。

- **按钮 `#btn-checkin`（“去打卡”）**
  - 逻辑样式：`px-4 py-3 rounded-2xl`。
  - 建议 1× 视觉尺寸：
    - 高度：与输入框齐平，**48px**。
    - 宽度：约 **112px**。
    - 圆角：**16px**。
  - 素材建议：
    - 出一张可横向拉伸的胶囊按钮背景（9-patch 思路），中间平滑，两端圆角。

### 2.3 地图图钉

- 当前原型图钉使用简单 DOM + Emoji，可选是否出专门贴图。
- 若出专用图钉素材：
  - 建议 1× 尺寸：**24 × 32px**（类“水滴”图钉）。
  - 需要两态：
    - 默认态：实心、颜色与主题一致。
    - 新打卡高亮态：带白色描边或发光。

## 3. 宠物房视图（view-petroom）

### 3.1 顶部信息区

- **标题栏区域（“宠物小屋” + 记忆按钮）**
  - 总高度：**56–64px**。
  - 左侧预留空位给返回或占位图标，不需额外素材。

- **标题文字 “宠物小屋”**
  - 使用 Cute 字体，前端渲染，无需切图。

- **记忆按钮 `#btn-memory`**
  - 逻辑尺寸：`w-10 h-10` → **40 × 40px** 圆形。
  - 建议素材：
    - 圆形按钮底板：1× 尺寸 **40 × 40px**（导出为 80×80@2x / 120×120@3x）。
    - 图标可继续用字体 Icon；如需 IP 风格独立图标，可出一枚“书本/记忆本”插画，直径 **20–24px**。

### 3.2 人格盲盒与称呼区域

- **人格盲盒状态条 `#personality-display`**
  - 容器：高度 **32–36px**，宽度 **280–320px** 左右。
  - 圆角：**16px**。
  - 背景：浅色卡片，可贴纹理但避免太复杂。

- **人格图标 `#p-icon`**
  - 逻辑尺寸：**26 × 26px** 圆形。
  - 建议导出：**52 × 52px@2x** / **78 × 78px@3x**。
  - 可根据不同人格做 3–4 款变体（小火苗/小云朵等）。

- **人格标签 `personality-tag`**
  - 字号：10px。
  - 胶囊高度：**18–20px**，左右内边距 4px。
  - 多个标签横向排布时注意留白。

- **人格盲盒按钮 `#btn-blind-box`**
  - 建议 1× 尺寸：
    - 高度：**28–32px**。
    - 最小宽度：**120px**。
    - 圆角：全圆角（9999px）。
  - 可出渐变背景条素材一张（胶囊形），内置礼物图标留白区。

- **称呼选项按钮 `.owner-title-option`**
  - 文本示例：“爸爸/妈妈/训练家/伙伴/铲屎官”。
  - 建议 1× 尺寸：
    - 高度：**22–24px**。
    - 最小宽度：**56–64px**。
    - 圆角：**9999px**。

- **自定义称呼输入框 `#owner-title-input`**
  - 宽度固定：`w-24` → **96px**。
  - 高度：**22–24px**。
  - 圆角：**9999px**。
  - 背景轻微描边+填充即可，无须复杂插画。

### 3.3 房间背景与布局

- **房间主背景 `#room-scene-bg`（当前为 `场景/场景.png`）**
  - 覆盖整个 `#room-viewport`，可水平略超出。
  - 建议原始插画尺寸：
    - 至少 **1920 × 1080**，更推荐 **2560 × 1440**。
  - 需要给以下区域留出清晰“可放东西”的平面：
    - 右下角：宠物站立位置。
    - 左侧：橱柜区域。
    - 墙面：挂画、日记、照片墙位置。

- **森林背景层 `#room-scene-forest-bg`**
  - 作为环境氛围层，尺寸更大（2× 视口）；建议：
    - 原始插画 ≥ **3000 × 2000**（可重复或模糊处理）。

- **照片墙 `#memory-wall`**
  - 相对房间背景的约束（以 1920 × 1080 为例）：
    - `top: 20%` → 约 **216px**。
    - `left: 10%` → 约 **192px**。
    - `width: 30%` → 约 **576px**。
    - `height: 25%` → 约 **270px**。
  - 每张小照片建议 1× 尺寸：**60 × 72px** 左右，带拍立得边框。

### 3.4 宠物主体与姿态

- **宠物容器 `#pet-avatar-room`**
  - 逻辑位置：`bottom: 20%`, `left: 60%`，在房间右下，略偏中间。
  - 容器本身会做轻微呼吸/行走动画，不需要额外插画。

- **宠物序列帧 `#pet-avatar-sprite`**
  - 显示尺寸：`w-24 h-24` → **96 × 96px**。
  - 素材规范（与 PRD 第 9.3 节对齐）：
    - 原始 1× 尺寸：**256 × 256px**，透明背景。
    - 建议导出：512 × 512@2x / 768 × 768@3x。
  - 动作类型：
    - 呼吸（idle_breath）。
    - 等待/张望（idle_wait / observe）。
    - 互动（interact）。
    - 休息/睡觉（rest）。
    - 吃东西（eat）。
    - 被摸（touch）。

### 3.5 房间交互热点

- 热点按钮统一使用 `.room-hotspot` / `.room-hotspot-button` 样式，结构为「Emoji + 文本」的小胶囊。

- **热点按钮 `room-hotspot-button`**
  - 1× 视觉尺寸：
    - 高度：**32–36px**。
    - 宽度范围：**52–80px**（文案：地图 / 日记 / 橱柜 / 小锅）。
  - 圆角：全圆角（9999px）。
  - 阴影：适中，带轻微发光（突出可点击性）。
  - Emoji 字号：约 20px（1.25rem），上方图标下方文字排布。
  - 若需出图：
    - 可出统一的“胶囊底板”素材，每个热点只换 Emoji 与文字。

## 4. 底部对话与能量指纹

### 4.1 底部对话栏 `#pet-dialog-bar`

- 位置：整体距离底部约 **24px**。
- 外容器宽度：`left-4 right-4` → 414 − 2 × 16 = **382px**。
- 文本区域：
  - 推荐最大宽度：**320–340px**，避免过长一行。
  - 圆角：**24px**。
  - 左侧预留一个 20×20px 图标区域（现用 💬，可换为统一小头像或气泡图标）。

### 4.2 能量指纹区 `#energy-fingerprint-area`

- 中心触控图标 `#energy-fingerprint-icon`
  - 1× 建议尺寸：
    - 可视直径：**40px**。
    - 点击热区：约 **64 × 64px**（由前端通过 padding 扩大，不必在图上画透明边）。
  - 样式建议：
    - 圆形指纹图案，带轻微发光。
    - 可提供按下态（亮度或阴影加强）。

### 4.3 底部提示文案 `petroom-bottom-hint`

- 字号：10px。
- 占用高度：**32px** 左右。
- 不需要额外插画，只需考虑文本与背景对比。

## 5. 橱柜视图与收集物插画

### 5.1 橱柜网格（隐藏容器 `#cabinet-grid`）

- 网格：`grid-cols-4 grid-rows-3`，共 12 个格子。
- 单格逻辑最小高度：`.cabinet-slot` `min-height: 100px`。
- 在 414 宽下，估算：
  - 内部实宽：约 414 − 2 × 16（左右 padding） − 3 × 8（列间距）≈ **350px**。
  - 单格宽度：约 **80–82px**。

### 5.2 橱柜槽位卡片 `.cabinet-slot`

- 1× 建议尺寸：
  - 内部高度：**100–110px**。
  - 宽度：**80–82px**（随屏幕略有浮动）。
  - 圆角：**16px**。
  - 边框：约 **2px** 浅色描边。
  - 阴影：轻微。

### 5.3 橱柜物件插画

- 媒体区域 `.cabinet-slot__media`
  - 高度：**52px**。
  - 宽度：与槽位同宽。
  - 插画置中显示。

- 插画素材规范：
  - 1× 目标展示尺寸：**最大边长约 52px**。
  - 建议原始尺寸：
    - 至少 **160 × 160px@2x** 或 **240 × 240px@3x**，保证缩小时细节清晰。
  - 形状：尽量在视觉上靠近正方形，以适应小方格。
  - 题材：按 PRD 里的地域配件（如：热干面碗、地标建筑小摆件等）。

## 6. 其他素材建议（与橱柜 / 日记联动）

### 6.1 场景缩略图（用于日记、橱柜联动）

- **场景缩略图**
  - 用途：用于日记卡片封面、小地图/特殊地点、橱柜中对应物件的联动展示。
  - 前端期望缩略图边长：**64–80px**。
  - 素材原始尺寸：至少 **256 × 256px**。
  - 构图：在正方形内居中，四角留出足够圆角裁切空间，避免关键信息被裁掉。

### 6.2 日记卡片与头像

- **宠物头像 / 圆形头像标识**
  - 用途：用于日记列表卡片左侧或顶部头像。
  - 显示直径：**32–40px**。
  - 建议与宠物序列帧复用同一头身设定，单独导出几帧静态头像（不同表情可选）。
- **日记卡片背景（如需要单独出图）**
  - 1× 建议尺寸：宽约 **320–350px**，高约 **80–120px**，圆角 **16–20px**。
  - 画面风格：纸张 / 便签感，保留足够文字区域，避免纹理过重影响可读性。

## 7. 本轮交付物清单（只改橱柜 + 日记相关）

本轮只需要输出与「橱柜视图」和「日记相关控件」有关的素材，其它页面可暂不出新稿：

1. **橱柜槽位底板与状态样式（可选）**
   - 如需要更统一的美术风格，可为 `.cabinet-slot` 出一张卡片底板（支持 12 个格子复用），包含正常态 / 选中态（高亮描边或发光）。
2. **橱柜物件插画**
   - 至少 **6–8 个代表性地区纪念品**（小方形图标），每个物件满足「1× 最大边长约 52px，原始 2×/3× 切图」规范。
3. **场景缩略图**
   - 若干枚方形缩略图，满足「边长 64–80px，原始 ≥256×256」规范，用于日记封面和橱柜联动展示。
4. **宠物头像 / 日记用头像**
   - 若干帧静态头像（不同表情可选），显示直径 32–40px，用于日记列表/详情中的头像。
5. **日记卡片背景（如采用贴图方案）**
   - 一到两种主题样式（例如：普通日记、重要日记），满足第 6.2 小节中的尺寸与风格约束。

画师在出稿时如需精确对齐某个区域，可直接按本规范中的 1× 尺寸在 414×896 画板内对位，或在有疑问时标注「预期在橱柜 / 日记中的大致位置」，便于前端微调。
